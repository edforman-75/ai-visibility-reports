How We Measured AI Visibility

Our approach to testing consumer AI surfaces

Overview

This report measures how often HUFT (Heads Up For Tails) appears in AI-generated responses to common pet product queries. We tested consumer-facing AI surfaces the way actual customers use them.

Surfaces Tested

We tested 4 consumer AI surfaces (browser) plus 3 API endpoints for a total of 7 distinct testing channels:

Surface Type Queries Tested Success Rate HUFT Mentions
ChatGPT Browser (chatgpt.com) 62 98% 34%
Perplexity Browser (perplexity.ai) 59 98% 43%
Gemini Browser (gemini.google.com) 59 98% 29%
Google AI Overviews Browser (google.com) 59 98% 22%
Gemini API API (2.0 Flash) 59 100% 68%
Perplexity API API (Sonar) 59 100% 42%
Llama 3.3 API (Together.ai) 59 100% 20%

Total queries executed: 416 across all surfaces

Key Finding: API vs Browser Differences

Gemini API (68%) far outperforms Gemini Browser (29%) for HUFT visibility. This suggests the browser interface applies additional filtering or context that affects brand mentions.

Perplexity shows consistent results between API (42%) and Browser (43%), indicating their interfaces are more closely aligned.

What We Measured

HUFT Mention Rate

The percentage of successful queries where HUFT or "Heads Up For Tails" appeared in the AI response. This is the primary metric for brand visibility.

Detection Criteria

A query was counted as a "HUFT mention" if the response contained either:

  • "HUFT" (case-insensitive)
  • "Heads Up For Tails" (case-insensitive)

Query Categories

We tested queries across multiple categories to understand visibility patterns:

  • Dog Food: General, premium, specialty diets
  • Cat Products: Food, litter, supplies
  • Accessories: Collars, beds, leashes
  • Grooming: Products, services
  • Shopping: Online stores, physical locations
  • Brand: Direct HUFT questions
  • Comparisons: HUFT vs competitors
  • Use Cases: New puppy, specific needs

Our Approach

Consumer-Perspective Testing

We tested AI surfaces through web browsers - the same way actual consumers use them. This captures the real-world experience including web search integration, personalization, and interface-specific behaviors.

Why Browser Testing Matters

API testing and browser testing often produce different results because:

  • Browser interfaces add web search capabilities
  • System prompts and configurations differ
  • Personalization and context affect responses
  • Post-processing and formatting vary

For a detailed explanation, see Browser vs API Analysis.

Limitations

Geographic Consideration

Tests were conducted from a non-India IP address. AI responses may differ when accessed from within India, as some surfaces use geographic context to customize responses.

Future improvement: Testing from an India-based IP address would provide more accurate results for what Indian consumers actually see. This is recommended for subsequent reports.

Other Limitations

  • Point-in-time snapshot: AI responses change over time as models are updated and web content evolves
  • Query sampling: We tested representative queries but cannot cover all possible variations
  • Language: All queries were in English; Hindi or regional language queries may yield different results
  • Personalization: Logged-in user experiences may differ from anonymous testing

Methodology Notes

Why We Don't Share Implementation Details

The specific techniques for accessing and testing AI surfaces are kept confidential for several reasons:

  • Surface interfaces change frequently; specific implementation details become outdated quickly
  • Detection methods could be used to game AI systems rather than improve genuine visibility
  • Our methodology represents ongoing research and development investment

What We Do Share

  • Surfaces tested: Which AI systems we evaluated
  • Query types: Categories and examples of questions asked
  • Metrics: How we calculated visibility scores
  • Results: Full visibility data by surface and category
  • Recommendations: Actionable steps to improve visibility

Recommended Improvements for Future Reports

1. India-Based IP Testing

Conduct tests from servers located in India to capture geo-targeted responses. This would ensure the results reflect what Indian consumers actually see.

2. Multi-Language Testing

Add Hindi and regional language queries to understand visibility in non-English contexts.

3. Logged-In Testing

Test with authenticated accounts to understand personalization effects.

4. Temporal Tracking

Run tests periodically to track visibility changes over time and measure the impact of optimization efforts.
